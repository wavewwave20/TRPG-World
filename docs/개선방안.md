## 2026-02-16 재발 방지 개선안 (오토스크롤 이슈)

### 문제 요약
- 증상: `판정 모달 닫힘 + 스토리 시작 직후`만 스크롤되어야 하는데, 스트리밍 완료 시점에도 하단 스크롤이 발생함.
- 원인 1: 스크롤 조건이 한 곳이 아니라 여러 트리거(시작 이벤트/초기 로드/세션 상태 변경)에 분산되어 있었음.
- 원인 2: effect 의존성이 `currentSession` 객체 단위여서, 동일 세션에서도 객체 갱신 시 초기 진입 로직이 재실행될 수 있었음.
- 원인 3: 이벤트 타이밍(`started` 이벤트의 늦은 도착 가능성) 방어 조건이 없었음.

### 코딩 규칙 (즉시 적용)
- 스크롤 정책은 "허용된 타이밍"만 명시하는 방식으로 작성한다.  
  예: `방 최초 진입 1회`, `판정 모달 닫힌 직후 + 시작 이벤트 1회`
- 실시간 화면 제어 로직은 객체 전체가 아니라 안정 키(`sessionId`) 기준으로 effect를 건다.
- 이벤트 기반 UI 액션에는 "시간/상태 게이트"를 둔다.  
  예: `modalClosedAt`이 최근 N초 이내일 때만 실행.
- 같은 동작을 여러 effect에서 동시에 수행하지 않는다.  
  스크롤 호출 지점을 단일 책임으로 제한한다.

### 작업 체크리스트 (수정 시 필수)
- [ ] 스크롤 호출 지점 전체 검색 (`scrollIntoView`, `window.scrollTo`) 후 의도 목록과 1:1 매핑 확인
- [ ] `started/complete/error` 이벤트 각각에서 스크롤 실행 여부를 명시적으로 표로 정리
- [ ] `currentSession` 등 객체 의존 effect를 `currentSessionId` 등 안정 값 의존으로 전환
- [ ] 모달/플래그 전환(열림→닫힘) 타이밍을 ref로 기록하고 이벤트 처리 시 검증
- [ ] "1회만 실행"이어야 하는 케이스는 실행 후 플래그 초기화

### QA 시나리오 (머지 전 필수)
1. 방 최초 입장: 로그 로드 후 하단으로 1회 이동
2. 판정 모달 닫힘 후 스토리 시작: 시작 직후 1회 이동
3. 스트리밍 진행 중: 추가 자동 스크롤 없음
4. 스트리밍 완료 시점: 자동 스크롤 없음
5. 동일 세션에서 상태 갱신(알림/참가자 변경 등): 초기 진입 스크롤 재실행 없음
6. 세션 재입장: 최초 입장 스크롤 정상 동작

### 리뷰 규칙 (PR 체크)
- "이벤트 타이밍/레이스"를 리뷰 항목으로 별도 체크한다.
- UI 상태 effect는 "왜 이 의존성이 필요한지" 코멘트를 남긴다.
- 재현 절차 + 수정 후 동작 영상을 PR 설명에 첨부한다.
