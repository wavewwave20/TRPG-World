## 1. 아키텍처 개요 (Architecture Overview)

- **Type:** Monolithic Setup (Frontend served separately or integrated)
- **Pattern:** Event-Driven Architecture (WebSocket 중심)
- **Key Constraint:** Low Latency (실시간 타자 효과 및 오디오 재생 필수)

---

## 2. Frontend (Client Side)

| **구분**                                                                         | **기술 스택**        | **선정 이유 (Decision Rationale)**                                                            |
| -------------------------------------------------------------------------------- | -------------------- | --------------------------------------------------------------------------------------------- |
| **Framework**                                                                    | **React (Vite)**     | - CSR(Client Side Rendering)에 최적화된 게임 앱이므로 Next.js보다 가벼운 Vite가 적합.         |
| - 컴포넌트 단위 개발(채팅창, 캐릭터창 분리)에 유리.                              |
| **Language**                                                                     | **TypeScript**       | - 복잡한 게임 상태(캐릭터 스탯 JSON, 채팅 로그 객체)의 **타입 안정성** 보장.                  |
| - 런타임 에러 방지.                                                              |
| **Styling**                                                                      | **Tailwind CSS**     | - 클래스명 고민 없이 빠르게 레이아웃(3-Pane Grid) 구축 가능.                                  |
| - 반응형 디자인 대응 용이.                                                       |
| **State Mgt**                                                                    | **Zustand**          | - Redux보다 훨씬 가볍고 보일러플레이트가 적음.                                                |
| - **실시간 데이터(Socket Event)**를 전역 상태에 반영하기에 구조가 매우 직관적임. |
| **Network**                                                                      | **Socket.io-client** | - WebSocket의 끊김 재연결, 룸(Room) 관리 등을 직접 구현할 필요 없이 라이브러리 차원에서 지원. |

---

## 3. Backend (Server Side)

| **구분**                                                                   | **기술 스택**       | **선정 이유 (Decision Rationale)**                                                            |
| -------------------------------------------------------------------------- | ------------------- | --------------------------------------------------------------------------------------------- |
| **Framework**                                                              | **FastAPI**         | - **Asyncio(비동기)** 기본 지원으로 WebSocket 처리에 파이썬 프레임워크 중 가장 성능이 뛰어남. |
| - Pydantic을 이용한 데이터 검증이 강력함.                                  |
| **Language**                                                               | **Python 3.10+**    | - AI 라이브러리(OpenAI, LangChain 등)와의 호환성 및 생태계가 가장 큼.                         |
| **Socket**                                                                 | **Python-SocketIO** | - FastAPI와 비동기 모드(ASGI)로 완벽하게 통합됨.                                              |
| - Room 관리, Broadcast 기능이 내장되어 있어 "방 별 게임 진행" 구현이 쉬움. |
| **Process**                                                                | **Uvicorn**         | - 비동기 Python 웹 서버(ASGI)의 표준. 빠르고 가벼움.                                          |

## 4. Database (Persistence)

| **구분**                                                                                                   | **기술 스택**                                     | **선정 이유 (Decision Rationale)**                                 |
| ---------------------------------------------------------------------------------------------------------- | ------------------------------------------------- | ------------------------------------------------------------------ |
| **RDBMS**                                                                                                  | **SQLite (Dev)**                                  |
| **PostgreSQL (Prod)**                                                                                      | - 개발 단계에선 파일 기반인 SQLite로 설정 간소화. |
| - 배포 시 **JSONB 타입**을 지원하는 PostgreSQL로 마이그레이션하여 **캐릭터 스탯(JSON)**을 효율적으로 쿼리. |
| **ORM**                                                                                                    | **SQLAlchemy**                                    | - Python의 표준 ORM. DB가 바뀌어도 코드를 수정할 필요가 없음.      |
| **Migration**                                                                                              | **Alembic**                                       | - DB 스키마 변경 이력을 관리하여 협업 및 배포 시 테이블 깨짐 방지. |

## 5. AI & Media (Intelligence)

| **구분**                                              | **기술 스택**                                                                            | **선정 이유 (Decision Rationale)**                           |
| ----------------------------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------ |
| **LLM**                                               | **OpenAI GPT-4o**                                                                        | - **JSON Mode** 지원이 강력하여 시스템 연동에 유리.          |
| - 한국어 처리 능력과 추론 속도가 현재 가장 균형 잡힘. |
| **TTS**                                               | **OpenAI TTS**                                                                           | - `alloy`, `echo` 등 TRPG에 어울리는 자연스러운 목소리 제공. |
| - 응답 속도가 빨라 실시간 스트리밍에 적합.            |
| **Streaming**                                         | **Server-Sent Events (SSE)**                                                             |
| or **Socket Stream**                                  | - AI가 생성하는 텍스트를 한 번에 받지 않고 토큰 단위로 받아 "타자 효과"를 구현하기 위함. |

## 6. 개발 도구 (Dev Tools)

- **Version Control:** Git & GitHub
- **API Testing:** Postman (HTTP), Firecamp (Socket 테스트용)
- **Package Mgt:**
  - Backend: `uv`
  - Frontend: `npm` or `yarn`
  - 통합배포: `docker`
