# TRPG World 서버 실행 가이드

## 사전 준비

### 필수 설치

- **Python** 3.11+
- **Node.js** 20.19+ (또는 22.12+)
- **uv** (Python 패키지 매니저)
- **npm** (Node 패키지 매니저)

### 환경 변수 설정

`backend/.env` 파일을 생성하고 아래 내용을 채워넣으세요:

```env
# 회원가입 시 필요한 접근 코드
REGISTRATION_CODE=your_access_code_here

# AI API 키 (둘 중 하나 이상 필수)
OPENAI_API_KEY=your_openai_api_key_here
GEMINI_API_KEY=your_google_api_key_here

# 사용할 LLM 모델 (기본값: gpt-4o)
# OpenAI: gpt-4o, gpt-4-turbo 등
# Gemini: gemini/gemini-3-pro-preview 등
LLM_MODEL=gpt-4o
```

## 로컬 개발 서버

### 1. 백엔드 실행

```bash
cd backend
uv sync                # 의존성 설치 (최초 1회)
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

정상 실행 시: `http://localhost:8000`

### 2. 프론트엔드 실행

```bash
cd frontend
npm install            # 의존성 설치 (최초 1회)
npm run dev
```

정상 실행 시: `http://localhost:5173`

> Vite 프록시가 설정되어 있어 `/api/*` 요청은 자동으로 백엔드(8000)로 전달됩니다.

### 3. 접속

브라우저에서 `http://localhost:5173` 접속

## Docker 배포

### 1. 환경 변수 설정

프로젝트 루트에 `.env` 파일 생성:

```env
# NPM(Nginx Proxy Manager) 네트워크 이름
NPM_NETWORK=nginx_proxy_manager_default

# CORS 허용 도메인
CORS_ORIGINS=https://your-domain.com,http://your-domain.com

# 회원가입 접근 코드
REGISTRATION_CODE=your_access_code_here

# AI API 키
OPENAI_API_KEY=your_openai_api_key_here
GEMINI_API_KEY=your_google_api_key_here
LLM_MODEL=gpt-4o
```

### 2. 빌드 및 실행

```bash
docker-compose up -d --build
```

### 3. 로그 확인

```bash
docker-compose logs -f backend     # 백엔드 로그
docker-compose logs -f frontend    # 프론트엔드 로그
```

### 4. 중지

```bash
docker-compose down
```

## 유용한 엔드포인트

| 경로 | 설명 |
|------|------|
| `GET /health` | 서버 상태 확인 |
| `GET /` | API 정보 |
| `POST /api/auth/register` | 회원가입 |
| `POST /api/auth/login` | 로그인 |
| `GET /api/sessions/` | 세션 목록 |
| `POST /api/sessions/` | 세션 생성 |
| `GET /api/characters/{id}` | 캐릭터 조회 |
| `POST /api/characters` | 캐릭터 생성 |

## 문제 해결

### 로그인 시 404 에러

Vite 프록시 설정이 누락된 경우 발생합니다. `frontend/vite.config.ts`에 프록시 설정이 있는지 확인하세요.

### DB 관련 에러

백엔드 최초 실행 시 SQLite DB 파일(`trpg_world.db`)이 자동 생성됩니다. 권한 문제가 있다면 `backend/` 디렉토리 쓰기 권한을 확인하세요.

### AI 응답 없음

`.env`의 API 키가 올바른지 확인하세요. `LLM_MODEL`에 맞는 키가 설정되어야 합니다:
- `gpt-4o` → `OPENAI_API_KEY` 필요
- `gemini/*` → `GEMINI_API_KEY` 필요
