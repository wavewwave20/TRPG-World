## 1. 개요 (Overview)

- **프로젝트 정의:** 호스트(Host)가 중재하고 AI가 서술하는 실시간 텍스트/오디오 TRPG.
- **아키텍처:** `FastAPI` (Backend) + `React` (Frontend) + `Socket.io` (Real-time)

---

## 3. 화면 UI 구조 (Layout Specs)

화면은 **3분할(Left, Center, Right)**로 구성되며, 호스트와 플레이어는 동일한 UI를 공유하되 **기능 활성화(Host Controls)** 여부만 다르다.

### 3.1. Left: 캐릭터 정보 (Status Pane)

- **기능:** 내 캐릭터의 실시간 상태 표시.
- **구성:**
  - `Stats View`: HP, MP, 주요 스탯 (JSON 기반 렌더링).
  - `Inventory`: 아이템 목록.
  - `World Info`: 버튼 클릭 시 현재 지역/상황 요약 모달 (AI Memory) 호출.

### 3.2. Right: 소통 창구 (General Chat Pane)

- **성격:** **휘발성(Ephemeral) 데이터**. DB에 영구 저장하지 않음
- **기능:**
  - `General Chat`: 플레이어 간 잡담, 작전 회의, AI 프롬프트에 포함되지 않음.
  - `System Log`: 플레이어 연결/끊김 알림, 주사위 결과 출력.

### 3.3. Center: 메인 스테이지 (Story Pane)

- **성격:** **영구(Persistent) 데이터**. DB `StoryLog` 테이블과 동기화됨.
- **구성:**
  - `Narrative View`: AI의 서술과 호스트가 확정한 행동 텍스트가 표시되는 곳.
  - `Typing Effect`: 실시간 타자 효과 적용.
  - `Action Input`: 플레이어(호스트 포함)가 행동을 입력하는 곳.
  - `Feedback`: 입력 시 "호스트 확인 중..." 상태 메시지 표시 (Input Disabled).

## 4. 핵심 프로세스 (Core Logic)

### 4.1. 행동 제출 및 중재 (Action Pipeline)

데이터 흐름: `Client` -> `Volatile Memory (Redis/RAM)` -> `Host Client`

1. **제출 (Submit):**
   - 플레이어들(호스트포함) 각자 행동을 입력.
   - 서버는 이를 DB에 저장하지 않고 메모리 큐(Queue)에 담아 호스트에게 소켓 이벤트 전송.
2. **중재 (Moderation - Host Only):**
   - 호스트 화면에 **[대기열 모달]** 팝업.
   - 호스트는 접수된 텍스트들을 수정(Edit)하거나 순서를 바꿔 최종 결정

### 4.2. 스토리 확정 및 저장 (Commit & Save)

데이터 흐름: `Host Client` -> `DB (StoryLog)` -> `AI Prompt`

1. **진행 (Proceed):** 호스트가 [전송] 버튼 클릭.
2. **DB 저장 (1차):** 확정된 문단을 `StoryLog` 테이블에 `role='USER'`로 저장.
3. **브로드캐스트:** 모든 플레이어의 중앙 화면에 확정된 문단 출력.
4. **AI 요청:** 저장된 내용을 포함하여 AI에게 생성 요청 트리거.

### 4.3. AI 생성 및 출력 (Generation)

데이터 흐름: `AI Model` -> `Stream` -> `Client` -> `DB (StoryLog)`

1. **스트리밍:** AI가 생성하는 텍스트를 실시간으로 중앙 화면에 출력.
2. **TTS 재생:** AI응답이 생성되면 TTS로 변환 후 재생 (모든 플레이어가 동일하게 재생되어야 함)
3. **DB 저장 (2차):** 생성이 완료되면 전체 응답을 `StoryLog` 테이블에 `role='AI'`로 저장.

## 5. 데이터베이스 스키마 (DB Schema)

**목표:** 복잡한 중간 과정은 버리고, 결과물만 심플하게 저장한다.

### `game_sessions` (방 정보)

| **Column**     | **Type** | **Description**                    |
| -------------- | -------- | ---------------------------------- |
| `id`           | PK       | 방 고유 번호                       |
| `host_user_id` | Int      | 호스트(방장) 유저 ID (권한 체크용) |
| `title`        | String   | 방 제목                            |
| `world_prompt` | Text     | AI 시스템 프롬프트 (세계관 설정)   |
| `ai_summary`   | Text     | AI 장기 기억용 요약본              |

### `characters` (캐릭터 시트)

| **Column**   | **Type** | **Description**                 |
| ------------ | -------- | ------------------------------- |
| `id`         | PK       | 고유 번호                       |
| `session_id` | FK       | 소속된 방                       |
| `user_id`    | Int      | 소유주 (호스트 or 플레이어)     |
| `name`       | String   | 캐릭터 이름                     |
| `data`       | JSON     | HP, MP, 아이템 등 유동적 데이터 |

### `story_logs` (메인 스토리)

| **Column**   | **Type** | **Description**                                          |
| ------------ | -------- | -------------------------------------------------------- |
| `id`         | PK       | 고유 번호                                                |
| `session_id` | FK       | 소속된 방                                                |
| `role`       | String   | **'USER'** (호스트가 정리한 행동) / **'AI'** (GM의 서술) |
| `content`    | Text     | 실제 표시되는 텍스트 내용(AI 응답)                       |
| `created_at` | Datetime | 정렬 순서                                                |
